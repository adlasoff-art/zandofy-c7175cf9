# Architecture

## System Overview

Zandofy operates as a multi-part platform:

- `frontend/` = React + Vite application, generated by Lovable, integrated and hardened in GitHub
- `backend/` = FastAPI API and business logic
- `frontend/supabase/` = Supabase migrations and Edge Functions
- `docker-compose.yaml` = deployment entry point used by Coolify

## Single Repository, Two Environments

Un seul dépôt GitHub sert les deux environnements. Coolify utilise deux ressources distinctes branchées sur le même repo :

- **Zandofy Staging** → branche `develop` → domaines `*-staging.zandofy.com`
- **Zandofy Production** → branche `main` → domaines `zandofy.com`, `api.zandofy.com`, etc.

Lovable et Cursor AI travaillent dans le même dépôt. Lovable pousse le frontend, Cursor intègre et ajuste le backend/DB/env. Tout est centralisé dans GitHub.

## Delivery Architecture

```text
Lovable
  -> scoped frontend code
  -> push vers GitHub (feature branch ou develop)

Cursor AI
  -> review, integration, backend, DB, env, deploy
  -> même repo GitHub

GitHub (adlasoff-art/Zandofy-project)
  -> single source of truth
  -> develop = staging
  -> main = production

Coolify Staging
  -> déploie develop
  -> studio-staging, api-staging, supabasa-staging

Coolify Production
  -> déploie main
  -> zandofy.com, api.zandofy.com, supabasa.zandofy.com
```

## Runtime Architecture

```text
Frontend (Vite build -> Nginx container)
  -> calls FastAPI backend
  -> calls Supabase public APIs

Backend (FastAPI)
  -> uses PostgreSQL / Supabase data
  -> business logic, auth, email, payment, media

Supabase
  -> database, auth, storage, functions
  -> instance staging séparée de l'instance production
```

## Sensitive Files

The following files are infrastructure-sensitive and must be edited carefully:

- `docker-compose.yaml`
- `docker-compose.prod.yml`
- `backend/Dockerfile`
- `frontend/Dockerfile`

## Architectural Rule

Feature work should not silently modify deployment, domains, ports, or variable naming. Those changes must be deliberate, documented, and reviewed.
